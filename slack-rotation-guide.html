全体の仕組み

① Slackの
ボタンを押す
→
② Slack が

→
③ GASが
スプレッドシートを確認
→

使うサービス役割費用
Slack（無料プランでも可）ボタンの表示・メンションの投稿無料
Googleスプレッドシート「今何番目か」を記録しておく無料
Google Apps Script（GAS）処理の中心。ボタンを受け取り動かす無料
補足：GAS（Google Apps Script）とは、Googleが無料で提供しているプログラム実行環境です。Googleアカウントがあれば誰でも使えます。サーバーを借りる必要はありません。

事前に準備するもの

✓Googleアカウント

スプレッドシートとGASを使うために必要です。すでに持っていればOKです。

✓メンション対象4人のSlack ユーザーID

メンションには @名前 ではなく ユーザーID（Uから始まる文字列） が必要です。

ユーザーIDの調べ方

  1. Slackでメンションしたい人のプロフィールを開く
  2. 「…」（その他）→「プロフィールをコピー」または「メンバーIDをコピー」をクリック
  3. U0123456789 のような文字列がコピーされます
メモしておきましょう：4人分のユーザーIDを事前にメモ帳などに控えておくとスムーズです。
✓ボタンを設置するSlackチャンネルのID

チャンネル名ではなくIDが必要です。

チャンネルIDの調べ方

  1. Slackでボタンを置きたいチャンネルを開く
  2. チャンネル名をクリック →「チャンネル詳細を開く」
  3. 一番下に C0123456789 のような文字列があります

STEP 1：Googleスプレッドシートを作る

1スプレッドシートを新規作成する

https://sheets.google.com を開き、「＋ 新しいスプレッドシート」をクリックします。

2シート名を変更する

画面下部のタブ「Sheet1」を右クリック →「名前を変更」→ State と入力してEnter。

3A1セルに「0」を入力する

A1セルをクリックして 0 と入力しEnter。これが「今何番目か」を記録するセルです。

意味：0=Aさんの番、1=Bさんの番、2=Cさんの番、3=Dさんの番 を表します。
4スプレッドシートのIDをメモする

ブラウザのURLを見てください。

https://docs.google.com/spreadsheets/d/ここがスプレッドシートID/edit

ハイライト部分（長い文字列）をコピーしてメモしておきます。

STEP 2：Google Apps Script を書く

1GASのエディタを開く

スプレッドシートのメニューから「拡張機能」→「Apps Script」をクリックします。

新しいタブでエディタが開きます。

2既存のコードを全部消す

最初から書いてある function myFunction() {} を全て選択して削除します。

3以下のコードをそのまま貼り付ける

★ 赤字のコメント箇所を自分の情報に書き換えてください。

// ===== ここを書き換えてください =====
const SHEET_ID = 'スプレッドシートのIDをここに貼る';
const CHANNEL_ID = 'SlackチャンネルのIDをここに貼る';

const MEMBERS = [
  { name: 'Aさん', id: 'AさんのSlackユーザーID' },
  { name: 'Bさん', id: 'BさんのSlackユーザーID' },
  { name: 'Cさん', id: 'CさんのSlackユーザーID' },
  { name: 'Dさん', id: 'DさんのSlackユーザーID' },
];
// ===================================

function getSlackToken() {
  return PropertiesService.getScriptProperties().getProperty('SLACK_TOKEN');
}

function doPost(e) {
  var body = e.postData.contents;

  // Slackの認証チェック（URLの確認用）
  try {
    var json = JSON.parse(body);
    if (json.type === 'url_verification') {
      return ContentService.createTextOutput(json.challenge)
        .setMimeType(ContentService.MimeType.TEXT);
    }
  } catch(e) {}

  // ボタンが押されたときの処理
  var decoded = decodeURIComponent(body.replace('payload=', ''));
  var payload = JSON.parse(decoded);

  if (payload.type === 'block_actions') {
    var lock = LockService.getScriptLock();
    lock.tryLock(5000);

    var sheet = SpreadsheetApp.openById(SHEET_ID).getSheetByName('State');
    var idx = Number(sheet.getRange('A1').getValue());
    var member = MEMBERS[idx];

    UrlFetchApp.fetch('https://slack.com/api/chat.postMessage', {
      method: 'post',
      contentType: 'application/json',
      headers: { 'Authorization': 'Bearer ' + getSlackToken() },
      payload: JSON.stringify({
        channel: CHANNEL_ID,
        text: '<@' + member.id + '> 担当です！',
      }),
    });

    sheet.getRange('A1').setValue((idx + 1) % MEMBERS.length);
    lock.releaseLock();
  }

  return ContentService.createTextOutput('OK');
}

function postInitialButton() {
  UrlFetchApp.fetch('https://slack.com/api/chat.postMessage', {
    method: 'post',
    contentType: 'application/json',
    headers: { 'Authorization': 'Bearer ' + getSlackToken() },
    payload: JSON.stringify({
      channel: CHANNEL_ID,
      text: '担当者メンションボタン',
      blocks: [{
        type: 'actions',
        elements: [{
          type: 'button',
          text: { type: 'plain_text', text: '次の担当者にメンション' },
          action_id: 'mention_next',
          style: 'primary',
        }],
      }],
    }),
  });
}
4Slack Tokenをスクリプトに登録する

セキュリティのため、TokenはコードにではなくGASの「プロパティ」に保存します。

  1. GASエディタの左メニュー「プロジェクトの設定」（歯車アイコン）をクリック
  2. 「スクリプト プロパティ」セクションで「プロパティを追加」をクリック
  3. プロパティ名：SLACK_TOKEN、値：後でSlack Appから取得するToken（xoxb-から始まる文字列）を入力
  4. 「スクリプトプロパティを保存」をクリック
注意：Slack Tokenはパスワードと同じです。コードに直接書いたり、人に見せたりしないでください。
Token取得タイミング：TokenはSTEP 4のSlack App作成後に取得します。先にGASを保存しておいて、Tokenは後から登録してもOKです。
5保存する

Ctrl+S（MacはCmd+S）またはフロッピーアイコンをクリックして保存します。

STEP 3：GAS をウェブアプリとして公開する

SlackからGASに通知を届けるため、GASをインターネット上に公開します。

1「デプロイ」をクリック

GASエディタ右上の青い「デプロイ」ボタン → 「新しいデプロイ」をクリックします。

2設定を行う
  1. 「種類の選択」→ 歯車アイコン → 「ウェブアプリ」を選択
  2. 説明（任意）：例 Slack Mention Workflow
  3. 「次のユーザーとして実行」→ 自分（自分のメールアドレス）
  4. 「アクセスできるユーザー」→ 全員
「全員」にする理由：Slackからのリクエストには認証情報がないため、誰でもアクセス可能にする必要があります。セキュリティはSlackの署名検証で担保します。
3デプロイして権限を許可する
  1. 「デプロイ」をクリック
  2. 「アクセスを承認」→ 自分のGoogleアカウントを選択
  3. 「このアプリはGoogleで確認されていません」と出たら「詳細」→「（プロジェクト名）に移動」をクリック
  4. 「許可」をクリック
4ウェブアプリURLをメモする

「デプロイが更新されました」画面に表示されるURLをコピーしてメモします。

https://script.google.com/macros/s/xxxxxxxxxxxxxxxxxx/exec
このURLは後でSlack Appに設定します。大切に保管してください。

STEP 4：Slack App を作成・設定する

1Slack API ページにアクセス

https://api.slack.com/apps を開きます。Slackにログインした状態でアクセスしてください。

2新しいAppを作成する
  1. 「Create New App」をクリック
  2. 「From scratch」を選択
  3. App Name：任意（例：順番メンション）
  4. Workspace：使用するSlackのワークスペースを選択
  5. 「Create App」をクリック
3権限（スコープ）を設定する

左メニュー「OAuth & Permissions」をクリック。

「Scopes」セクションの「Bot Token Scopes」で「Add an OAuth Scope」をクリックし、以下を追加：

スコープ名用途
chat:writeメッセージ・メンションを投稿する
4Appをワークスペースにインストールする
  1. 同じ「OAuth & Permissions」ページの上部「Install to Workspace」ボタンをクリック
  2. 「許可する」をクリック
  3. 「Bot User OAuth Token」（xoxb-で始まる）が表示されるのでコピーする
このTokenをGASのスクリプトプロパティ（SLACK_TOKEN）に登録してください。（STEP 2-4で空欄にしていた場合は今登録します）
5Interactivity（ボタン操作の受信）を有効にする

左メニュー「Interactivity & Shortcuts」をクリック。

  1. 「Interactivity」トグルを ON にする
  2. 「Request URL」にSTEP 3でメモしたGASのURLを貼り付ける
  3. 右下「Save Changes」をクリック
6BotをSlackチャンネルに招待する

Slackでボタンを置くチャンネルを開き、次のメッセージを送信します：

/invite @順番メンション

（作成したAppの名前に合わせて変えてください）

STEP 5：Slackにボタンを投稿する

GASからボタンを含むメッセージをSlackに1回だけ投稿します。

1GASエディタで関数を実行する
  1. GASエディタを開く
  2. 画面上部の関数選択ドロップダウンで postInitialButton を選択
  3. 「▶ 実行」ボタンをクリック
初回のみ：権限の確認が出たら「許可」してください。
2Slackを確認する

指定したチャンネルに「次の担当者にメンション」というボタンが投稿されていれば成功です。

ボタンはこの1回だけ投稿すればOKです。メッセージは削除しないようにしましょう。

STEP 6：動作テスト

1ボタンを押してみる

Slackのボタンをクリックしてください。

  • 1回目：Aさんにメンションされる
  • 2回目：Bさんにメンションされる
  • 3回目：Cさんにメンションされる
  • 4回目：Dさんにメンションされる
  • 5回目：Aさんに戻る
2スプレッドシートも確認する

A1セルの数字がボタンを押すたびに 0 → 1 → 2 → 3 → 0 … と変わっていれば正常です。

3担当者をリセットしたい場合

スプレッドシートのA1セルを 0 に書き換えるだけでAさんからリスタートできます。

困ったときは

症状確認ポイント
ボタンを押しても何も起きない ・GASのRequest URLが正しく設定されているか
・GASが「全員」アクセス可能でデプロイされているか
・Bot TokenがGASのプロパティに登録されているか
メンションが届かない ・BotがチャンネルにInviteされているか
・SlackユーザーIDが正しいか（Uから始まっているか）
・チャンネルIDが正しいか（Cから始まっているか）
「not_in_channel」エラー ・/invite @ボット名 でBotをチャンネルに招待する
「invalid_auth」エラー ・GASプロパティのSLACK_TOKENが正しく設定されているか確認
コードを変更したのに反映されない ・GASを変更したら「デプロイ」→「デプロイを管理」→「新しいバージョン」で再デプロイが必要
同じ人が連続でメンションされる ・スプレッドシートのA1セルが正しく更新されているか確認

GASのログを確認する方法

エラーの原因を調べたいときは、GASエディタの左メニュー「実行数」をクリックすると処理の記録が見られます。

担当者の人数を変えたい場合：GASコードの MEMBERS 配列に追加・削除するだけでOKです。変更後は必ず再デプロイしてください。
